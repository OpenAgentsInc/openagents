# OpenAgents Engine Technical Specification

## Overview

The OpenAgents Engine is a local web-based autonomous agent system designed to process GitHub issues and perform actions based on their content. The system uses Effect.js for functional programming patterns, Anthropic's Claude API for AI analysis, and a streaming architecture for real-time updates.

## Architecture

### Core Components

1. **Server**: Web server that handles HTTP requests and provides SSE (Server-Sent Events) for real-time updates
2. **Program**: Main application logic that processes GitHub issues using AI
3. **GitHub Client**: Service for interacting with GitHub API
4. **GitHub Tools**: AI tools for GitHub operations
5. **Agent State Management**: System for tracking and updating agent state

### Technical Stack

- **Runtime**: Node.js
- **Functional Programming**: Effect.js
- **AI Integration**: Anthropic Claude API via @effect/ai-anthropic
- **Web Interface**: HTMX for simple client-side interactions
- **Real-time Updates**: Server-Sent Events (SSE)
- **Type Safety**: TypeScript with strict mode

## Detailed Component Specifications

### 1. Server Component (`Server.ts`)

The server component provides HTTP endpoints and SSE event streaming for client interaction.

#### Key Features

- HTTP server that serves static files from the `/public` directory
- SSE event stream for real-time updates to clients
- Endpoints for processing GitHub issues
- Environment variable configuration

#### Endpoints

- **GET /**: Serves the main web interface
- **GET /events**: SSE endpoint for real-time updates
- **POST /api/process**: Processes a GitHub issue

#### SSE Events

The server emits the following events:
- `connection`: Status of server connection
- `analysis`: Updates about GitHub issue analysis
- `agent_state`: Current state of the agent
- `error`: Error information when issues occur

### 2. Program Component (`Program.ts`)

The program component contains the main application logic for processing GitHub issues.

#### Key Features

- GitHub issue analysis using Claude AI
- Tool integration for GitHub operations
- Streaming responses for real-time updates
- Configuration via environment variables

#### Main Functions

- `processGitHubIssue`: Core function that processes a GitHub issue
- Integration with Claude AI via `completions.toolkitStream`
- Response processing via Effect.js pipelines

#### Configuration

- `GITHUB_REPO_OWNER`: Owner of the GitHub repository
- `GITHUB_REPO_NAME`: Name of the GitHub repository
- `GITHUB_ISSUE_NUMBER`: Issue number to process
- `ANTHROPIC_API_KEY`: API key for Anthropic Claude

### 3. GitHub Client (`GitHub.ts`)

The GitHub client provides a service for interacting with the GitHub API.

#### Key Features

- HTTP client for GitHub API with proper authentication
- Methods for GitHub issue operations
- Agent state storage and retrieval
- Error handling and logging

#### GitHub API Operations

- `getIssue`: Get issue details by number
- `listIssues`: List issues in a repository
- `getIssueComments`: Get comments on an issue
- `createIssueComment`: Create a comment on an issue
- `getRepository`: Get repository information
- `updateIssue`: Update issue properties (title, body, state, labels, assignees)

#### Agent State Operations

- `saveAgentState`: Save agent state to local storage
- `loadAgentState`: Load agent state from local storage
- `createAgentStateForIssue`: Create a new agent state for an issue

### 4. GitHub Tools (`GitHubTools.ts`)

The GitHub tools component provides AI tools for GitHub operations.

#### Key Features

- AI tools for GitHub operations using @effect/ai
- Tool definitions with parameters and descriptions
- Tool implementations with error handling
- Service layer for tool access

#### Tool Definitions

- `GetGitHubIssue`: Get a GitHub issue by number
- `ListGitHubIssues`: List GitHub issues for a repository
- `CreateGitHubComment`: Create a comment on a GitHub issue
- `UpdateGitHubIssue`: Update a GitHub issue
- `GetGitHubRepository`: Get information about a GitHub repository
- `GetGitHubIssueComments`: Get comments on a GitHub issue
- `CreateAgentStateForIssue`: Create a new agent state for an issue
- `LoadAgentState`: Load an existing agent state
- `SaveAgentState`: Save the current agent state

### 5. GitHub Types (`GitHubTypes.ts`)

The GitHub types component provides type definitions for GitHub API responses and agent state.

#### Key Features

- Type definitions for GitHub API responses
- Agent state type definitions
- Schema validation using Effect.js Schema system

#### GitHub API Types

- `GitHubUser`: GitHub user information
- `GitHubLabel`: GitHub label information
- `GitHubIssue`: GitHub issue information
- `GitHubIssueComment`: GitHub issue comment information
- `GitHubRepository`: GitHub repository information
- `GitHubIssueListResponse`: Response for listing issues
- `GitHubError`: Error information from GitHub API

#### Agent State Types

- `AgentInfo`: Basic agent information (type, version, ID)
- `Timestamps`: Timestamps for agent actions
- `IssueDetailsCache`: Cached issue details
- `CurrentTask`: Current task information
- `ToolCall`: Tool call information
- `PlanStep`: Step in the agent's plan
- `ExecutionContext`: Context for execution
- `ConversationMessage`: Message in a conversation
- `KeyDecision`: Key decision made by the agent
- `ImportantFinding`: Important finding by the agent
- `Memory`: Agent memory information
- `Metrics`: Agent performance metrics
- `ErrorState`: Error state information
- `LLMConfig`: LLM configuration
- `Configuration`: Agent configuration
- `AgentState`: Overall agent state

## Data Flow

### Issue Processing Flow

1. User enters GitHub repository and issue information
2. Server receives processing request
3. GitHub client fetches issue information
4. AI analyzes the issue with GitHub tools
5. Agent state is created and updated
6. Real-time updates are sent via SSE
7. UI displays the analysis results

### Agent State Management Flow

1. Agent state is created for a GitHub issue
2. Agent performs analysis and updates state
3. Agent state is saved to local storage
4. UI displays the current agent state
5. Agent can load state for continued processing

## Error Handling

### Error Types

- `HttpClientError`: Errors from HTTP clients

- `AiError`: Errors from AI processing

### Error Handling Strategies

1. **GitHub API Rate Limits**
   - Standard GitHub API rate limits apply
   - Potential for throttling during heavy usage

3. **Local Storage Constraints**

4. **Error Recovery**

   - Manual intervention may be required for some errors

## Technical Implementation Details

### Effect.js Patterns

The application extensively uses Effect.js patterns for functional programming:

1. **Services and Layers**
   - Services defined with `Effect.Tag`
   - Layers for dependency injection
   - Layer composition with `Layer.provide`

2. **Effect Composition**
   - Pipeline composition with `pipe`
   - Generator functions with `yield*`
   - Effect transformations with `flatMap`, `map`, etc.

3. **Error Handling**
   - Type-safe error handling with `Effect.catchAll`
   - Error logging with `Effect.tapError`
   - Error transformation with `Effect.mapError`

4. **Schema Validation**
   - Schema definitions with `Schema.Class`
   - Request/response validation
   - Type-safe parsing

### AI Integration

The application integrates with Anthropic Claude for AI capabilities:

1. **AI Tools**
   - Tool definitions with `AiToolkit`
   - Tool implementations with handlers
   - Tool streaming with `toolkitStream`

2. **Prompt Engineering**
   - Task-specific prompts for GitHub issue analysis
   - System prompts for agent behavior
   - Tool usage instructions

3. **Response Processing**
   - Stream processing of AI responses
   - Chunk processing for real-time updates
   - Tool call handling and result incorporation

### HTTP and Web Integration

The application uses HTTP and web technologies for client interaction:

1. **HTTP Server**
   - HTTP request handling
   - Static file serving
   - API endpoints

2. **SSE Streaming**
   - Event stream creation
   - Event type differentiation
   - Client connection management

3. **HTMX Integration**
   - HTML updates with HTMX
   - Event-triggered UI updates
   - Minimal client-side JavaScript
