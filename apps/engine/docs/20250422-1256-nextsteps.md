# Next Steps: Migrating and Updating Tests

## Overview

This document outlines the remaining tasks to complete the migration from the "overnight" branch to the "engine" app. The primary focus is on moving and updating tests to ensure full test coverage of the GitHub integration.

## Tasks for Next Agent

1. **Locate and migrate missing tests** from the "overnight" branch:
   - Examine tests in the original branch that may not have been migrated yet
   - Pay special attention to integration tests that may be testing multiple components together
   - Look for test fixtures and test utilities that may need to be migrated

2. **Update existing tests** to ensure they properly test the new functionality:
   - Enhance `GitHub.test.ts` to test all GitHub client methods
   - Expand `GitHubTools.test.ts` to cover all GitHub tools
   - Add tests for agent state management functionality

3. **Add integration tests**:
   - Create tests that verify the end-to-end functionality
   - Test the GitHub issue processing workflow
   - Test the SSE event streaming
   - Test error handling scenarios

4. **Update mock implementations**:
   - Enhance the GitHub API mocks to provide realistic responses
   - Create fixtures for GitHub issues, comments, and repositories
   - Implement mock Claude AI responses for testing

5. **Test coverage analysis**:
   - Run test coverage report to identify gaps in test coverage
   - Prioritize adding tests for uncovered code paths
   - Focus on critical error handling paths

6. **Test performance scenarios**:
   - Add tests for concurrent GitHub operations
   - Test handling of rate limiting
   - Test with large GitHub issues and comments

## Implementation Guidelines

1. **Test organization**:
   - Keep tests in `/test/github/` directory
   - Name test files to match the modules they test
   - Group tests by functionality

2. **Testing strategy**:
   - Use unit tests for individual functions and methods
   - Use integration tests for component interactions
   - Use mock HTTP responses for GitHub API

3. **Test utilities to create**:
   - Fixtures for GitHub API responses
   - Helper functions for creating test agent states
   - Mock implementations for testing error cases
