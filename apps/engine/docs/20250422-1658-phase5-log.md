# Phase 5 - State Integration with UI

## Implementation Plan

Phase 5 focuses on integrating the implemented state management services into the main server workflow and updating the UI to display the agent state information in real-time. The key components to implement are:

1. Refactor `Server.ts` to use the agent state management services
2. Add execution loop in the `/fetch-issue` endpoint handler
3. Broadcast agent state updates via SSE
4. Update `index.html` to display agent state information

## Implementation Components

### 1. Server.ts Modifications
- Added imports for all required services and layers
- Modified `/fetch-issue` handler to:
  - Create an execution loop using Effect patterns
  - Load or create agent state using `GitHubClient` service
  - Execute steps using `TaskExecutor` service
  - Broadcast state updates via a new `agent_state` SSE event
  - Handle errors and termination conditions
- Implemented proper dependency injection with layers

### 2. Index.html Updates
- Added new HTML elements to display agent state information
- Added JavaScript to handle the `agent_state` SSE events
- Improved the status display section
- Implemented progress tracking for plan steps

## Implementation Details

### Server.ts Implementation

The core changes to `Server.ts` include:
- Using a `Ref` to maintain the current state throughout the execution loop
- Creating a recursive execution pattern using `Effect.suspend` to run steps sequentially
- Broadcasting state updates after each step execution
- Handling terminal states to break the execution loop
- Proper error handling for step execution failures

### UI Updates

The UI changes focus on:
- Adding structured HTML elements to display state information
- Implementing client-side JavaScript to handle SSE events
- Displaying current step, progress, status, and error information
- Ensuring real-time updates as the agent processes tasks

## Verification

The implementation was verified by:
- Type checking with `pnpm check`
- Testing the UI and server interaction
- Confirming state updates are properly displayed in the UI
- Validating proper error handling and state transitions

## Challenges and Decisions

1. **SSE Event Format**: Decided to use JSON for state updates to maintain flexibility
2. **UI Update Strategy**: Chose to implement a single state update event rather than multiple event types to simplify the UI handling
3. **Execution Flow**: Implemented a recursive pattern for step execution to ensure proper sequencing and state updates
4. **Error Handling**: Added comprehensive error handling to ensure the UI reflects failure states correctly
