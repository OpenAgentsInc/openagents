## Implementation Details

### 1. Adding StatefulToolContext to GitHubTools.ts

I created a new `StatefulToolContext` interface in `GitHubTools.ts` with the following structure:

```typescript
export interface StatefulToolContext {
  readonly stateRef: Ref.Ref<AgentState>
  readonly planManager: PlanManager
  readonly memoryManager: MemoryManager
}

export const StatefulToolContext = Context.Tag<StatefulToolContext>()
```

This context provides access to:
- A mutable reference to the current `AgentState`
- The `PlanManager` for adding tool calls to steps
- The `MemoryManager` for adding tool invocations to logs

### 2. Refactoring Tool Handlers

Each tool handler in `GitHubTools.ts` was updated to:
1. Access the `StatefulToolContext` using `yield* StatefulToolContext`
2. Perform the core GitHub API action
3. Update the state via the `stateRef` to record the tool invocation, using both `memoryManager` and `planManager`
4. Update metrics (tools called count)
5. Handle errors properly, updating error state

Example pattern:
```typescript
Effect.gen(function*() {
  const { stateRef, memoryManager, planManager } = yield* StatefulToolContext
  
  // Core API action
  const result = yield* github.someAction(params)
  
  // Update state after success
  yield* Ref.update(stateRef, (currentState) => {
    // Create tool call data entry
    // Add to logs, metrics, and plan step
    // Return updated state
  })
  
  return result
}).pipe(
  Effect.catchAll((error) => 
    Effect.gen(function*() {
      // Update state with error information
      yield* Ref.update(stateRef, (state) => { /* error state updates */ })
      return yield* Effect.fail(new ToolExecutionError(...))
    })
  )
)
```

### 3. Updating TaskExecutor.ts

I completely replaced the simulated AI solution with real AI integration:

1. Added imports for `Completions` and `AnthropicCompletions`
2. Modified `TaskExecutorLayer` to get `completions` from the context
3. Replaced `simulateAIWithTools` with real AI `toolkitStream`
4. Added state management for the AI stream processing:
   - Handling text chunks from Claude
   - Processing tool calls and their results
   - Updating agent state appropriately

Main structure:
```typescript
// Create state ref and tool context
const stateRef = yield* Ref.make(workingState)
const toolContextService = { stateRef, planManager, memoryManager }
const toolContextLayer = Layer.succeed(StatefulToolContext, toolContextService)

// Get AI response stream
const aiResponseStream = completions.toolkitStream({
  model: workingState.configuration.llm_config.model,
  messages: [{ role: "user", content: prompt }],
  tools: { toolkit: tools, handlers }
  // ...other options
})

// Process the stream
yield* aiResponseStream.pipe(
  Stream.tap((chunk) => /* process responses & tool calls, update state */),
  (stream) => Effect.provide(stream, toolContextLayer),
  Stream.runDrain
)
```
