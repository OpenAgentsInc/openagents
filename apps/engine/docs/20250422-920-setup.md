# 20250422-920 Initial Server Setup

## Changes Made

### 1. Added Dependencies
Added the required dependencies for the server implementation:
- `@effect/platform`: Core platform functionality
- `@effect/platform-node`: Node.js specific platform functionality

### 2. Implemented HTTP and WebSocket Server
Created a server implementation with:
- A root route (`/`) that serves HTML content with:
  - Black background
  - White monospace text
  - "Hello world" message
  - Content served from `public/index.html`
- A WebSocket endpoint at `/ws` that:
  - Accepts WebSocket upgrade requests
  - Sends a "Connected to WebSocket" message on connection
  - Implements ping-pong functionality (responds with "pong" when sent "ping")

### 3. Created Directory Structure
- `public/` - Contains static files to be served
  - `index.html` - The HTML file served at the root route

### 4. Implementation Details
- Used Effect.js for the main application structure
- Used Node.js built-in `http` module for the HTTP server
- Served static HTML from public/index.html file
- Added robust file path resolution using process.cwd() instead of __dirname
- Implemented comprehensive logging for debugging and monitoring
- Implemented raw WebSocket protocol handling for the WebSocket endpoint
- Created a simple but fully functional WebSocket frame parser/generator for the ping-pong functionality
- Added error handling throughout the application
- Integrated with HTMX WebSocket extension for real-time UI updates

### 5. HTMX Integration
- Used the HTMX WebSocket extension (ws.js) to handle WebSocket communication
- Implemented Out-of-Band (OOB) swaps for updating different parts of the page
- Server responds with HTML fragments that HTMX automatically processes
- Each WebSocket message is structured as HTML with specific target IDs:
  ```html
  <div id="messages" hx-swap-oob="beforeend">...</div>
  <div id="response" hx-swap-oob="innerHTML">...</div>
  ```
- This approach allows for server-driven UI updates without custom JavaScript

## Implementation Approach
Initially, I attempted to use the more idiomatic Effect.js HTTP and WebSocket APIs, but encountered compatibility issues with the current version of the dependencies. To ensure a working solution, I used a hybrid approach:

1. Effect.js for the overall program structure and lifecycle
2. Node.js's built-in `http` module for the HTTP server
3. Manual WebSocket protocol handling for the WebSocket endpoint

This approach allowed me to implement the required functionality while maintaining the Effect.js pattern for the overall application.

## Testing the Implementation

### HTTP Endpoint
1. Start the server with `pnpm start`
2. Open a browser and navigate to http://localhost:3000
3. Verify that the WebSocket test page appears with black background and white monospace text

### WebSocket with HTMX Integration
1. Start the server with `pnpm start`
2. Open a browser and navigate to http://localhost:3000
3. The page will automatically connect to the WebSocket endpoint at ws://localhost:3000/ws
4. You'll see a "Connected to WebSocket" message in the message container
5. Click the "Send Ping" button to send a ping message
6. The server will respond with a "PONG!" message that appears in the response area
7. You can also type custom messages in the text input and click "Send" to see the server response

### Testing with External Tools
You can also test the WebSocket endpoint using tools like wscat:
```
npm install -g wscat
wscat -c ws://localhost:3000/ws
```
- After connecting, you'll receive the message "Connected to WebSocket"
- Type "ping" and you'll receive "pong" in response
