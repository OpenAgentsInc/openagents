# Plan: Script-Based Agent Execution Workaround

## 1. Objective

To bypass the persistent runtime `Service not found: PlanManager` error occurring within the `Server.ts` `Effect.runFork` context, enabling development and testing of the core agent state management, AI interaction, and tool execution logic in a controlled, non-server environment.

## 2. Rationale

Debugging the root cause of the service resolution failure within the `Server.ts`/`Effect.runFork` context is proving extremely difficult and time-consuming. The core agent logic (`TaskExecutor`, state managers, tools, AI integration) needs to be validated independently of the web server integration complexities. Running the agent workflow as a standalone script will:

1.  **Isolate Core Logic:** Separate the agent's execution engine from potential issues related to HTTP requests, SSE, `Effect.runFork`, or complex context propagation in the server environment.
2.  **Simplify Debugging:** Provide a more direct execution flow, making it easier to trace state changes and pinpoint errors within the agent's decision-making loop.
3.  **Unblock Development:** Allow progress on implementing and testing features like error handling, retry logic (Phase 4, Step 12), and integration tests (Phase 4, Step 13) without being blocked by the server runtime issue.
4.  **Validate `AllLayers`:** Confirm whether the `AllLayers` composition works correctly in a different execution context (`Effect.runPromise`).

## 3. Implementation Plan: `RunAgentScript.ts`

A new entry point script (`src/RunAgentScript.ts`) will be created to orchestrate the agent's execution for a single issue.

**3.1. Script Structure (`src/RunAgentScript.ts`):**

```typescript
import { Effect, Layer, Console } from "effect";
import { AllLayers } from "./Program.js"; // Use the single source of truth layer
import { GitHubClient } from "./Program.js"; // Import Tags from Program
import { TaskExecutor } from "./Program.js"; // Import Tags from Program
import type { AgentState } from "./github/AgentStateTypes.js";

// --- Configuration ---
const config = {
    owner: process.env.RUN_SCRIPT_OWNER || "openagentsinc",
    repo: process.env.RUN_SCRIPT_REPO || "openagents",
    issueNumber: parseInt(process.env.RUN_SCRIPT_ISSUE || "796", 10),
    // Optional: Load existing state instance ID?
    // instanceId: process.env.RUN_SCRIPT_INSTANCE_ID || undefined
};
// Ensure API Keys are loaded (dotenv should be called early by Program.js/index.js import)
if (!process.env.ANTHROPIC_API_KEY || !process.env.GITHUB_TOKEN) {
    console.error("Error: ANTHROPIC_API_KEY and GITHUB_TOKEN must be set in environment.");
    process.exit(1);
}
// --- End Configuration ---

// --- Core Agent Execution Effect ---
const agentExecutionPipeline = Effect.gen(function*() {
    Console.log(`Starting agent execution for ${config.owner}/${config.repo}#${config.issueNumber}`);

    const githubClient = yield* GitHubClient;
    const taskExecutor = yield* TaskExecutor;

    // 1. Load or Create Initial State
    // TODO: Add logic to load state if config.instanceId is provided
    const initialState: AgentState = yield* githubClient.createAgentStateForIssue(
        config.owner,
        config.repo,
        config.issueNumber
    ).pipe(
        Effect.tap((state) => Console.log(`Created initial state with ID: ${state.agent_info.instance_id}`)),
        Effect.catchAll((error) => {
            Console.error("Failed to create initial state:", error);
            return Effect.fail("InitialStateCreationError");
        })
    );

    let currentState = initialState;

    // 2. Execution Loop
    let maxSteps = 20; // Safety break
    let stepsRun = 0;
    while (stepsRun < maxSteps) {
        stepsRun++;
        Console.log(`\n--- Executing Step ${currentState.current_task.current_step_index + 1} (Run ${stepsRun}) ---`);

        // Check for terminal status before execution
        const status = currentState.current_task.status;
        if (status === "completed" || status === "error" || status === "blocked") {
            Console.log(`Execution loop terminated. Final Status: ${status}`);
            if (status === "error" || status === "blocked") {
                 Console.log("Error State:", currentState.error_state);
            }
            break;
        }

        // Check if plan index is valid
         if (currentState.current_task.current_step_index >= currentState.plan.length) {
             Console.log(`Execution loop terminated. Reached end of plan.`);
             // Optionally update status to completed if not already error/blocked
             currentState = { ...currentState, current_task: { ...currentState.current_task, status: "completed" } };
             yield* githubClient.saveAgentState(currentState); // Save final state
             break;
         }

        // Execute the next step
        const nextState = yield* taskExecutor.executeNextStep(currentState).pipe(
            Effect.catchAll((error) => {
                Console.error(`Runtime error during executeNextStep (Loop ${stepsRun}):`, error);
                // Attempt to save state even after error, potentially updating error fields
                const errorState = {
                    ...currentState, // Use state *before* the failed step attempt
                    current_task: { ...currentState.current_task, status: "error" as const },
                    error_state: {
                        ...currentState.error_state,
                        last_error: { timestamp: new Date().toISOString(), message: `Unhandled runtime error: ${error}`, type: "internal" as const, details: String(error) },
                        consecutive_error_count: currentState.error_state.consecutive_error_count + 1
                    }
                };
                return githubClient.saveAgentState(errorState).pipe(
                    Effect.flatMap(() => Effect.fail(`ExecuteStepError-${stepsRun}`)) // Fail the pipeline after saving
                );
            })
        );
        currentState = nextState; // Update state for the next iteration
    }

    if (stepsRun >= maxSteps) {
        Console.log(`Execution loop terminated after reaching max steps (${maxSteps}).`);
    }

    Console.log(`\n--- Agent Execution Finished ---`);
    Console.log(`Final State ID: ${currentState.agent_info.instance_id}`);
    Console.log(`Final Status: ${currentState.current_task.status}`);
    // Log other relevant final metrics or state details if desired

    return currentState; // Return final state
});
// --- End Core Agent Execution Effect ---


// --- Run the Script ---
Console.log("Providing AllLayers and running the agent execution pipeline...");

Effect.provide(agentExecutionPipeline, AllLayers).pipe(
    Effect.tapBoth({
        onFailure: (error) => Console.error(`Pipeline Failed: ${error}`),
        onSuccess: (finalState) => Console.log(`Pipeline Succeeded. Final Status: ${finalState.current_task.status}`)
    }),
    Effect.runPromise // Execute the entire pipeline as a Promise
).then(() => {
    Console.log("Script finished.");
    process.exit(0);
}).catch(err => {
    Console.error("Unhandled Promise rejection:", err);
    process.exit(1);
});
// --- End Run the Script ---
```

**3.2. Modifications Needed:**

*   **Environment Variables:** The script will rely on environment variables (`RUN_SCRIPT_OWNER`, `RUN_SCRIPT_REPO`, `RUN_SCRIPT_ISSUE`, `ANTHROPIC_API_KEY`, `GITHUB_TOKEN`). Ensure these are set or have sensible defaults. The `.env` loading mechanism used by `Server.ts`/`Program.ts` should ideally be triggered by the imports.
*   **State Loading:** Add logic to optionally load a pre-existing state file if an `instanceId` is provided via `RUN_SCRIPT_INSTANCE_ID`.
*   **Error Handling:** Enhance error handling within the loop and the main `runPromise` catch block.
*   **Terminal Conditions:** Refine the `while` loop termination conditions.

**3.3. Execution:**

1.  Save the code above as `src/RunAgentScript.ts`.
2.  Add a script to `package.json`:
    ```json
    "scripts": {
      // ... other scripts ...
      "run-agent": "pnpm build-esm && node build/esm/RunAgentScript.js"
    },
    ```
3.  Ensure API keys and target issue details are set in `.env` or the environment.
4.  Run: `pnpm run-agent`

**3.4. Success Criteria:**

*   The script executes without `Service not found` errors.
*   Logs indicate the agent progressing through plan steps.
*   AI calls are made (verify via logs in `TaskExecutor`).
*   Tool calls are made and logged in the state (verify via logs/state file).
*   The script terminates gracefully (either reaches end of plan, max steps, or a defined terminal state like `completed`/`error`/`blocked`).
*   The final `state/{instance_id}.json` file reflects the execution history.

**3.5. Next Steps:**

*   **If Successful:** Analyze the execution flow and final state. Proceed with implementing error handling/retries (Step 12) and integration tests (Step 13) using this script-based approach. Once the core logic is robust, revisit the `Server.ts` integration, potentially applying lessons learned.
*   **If Fails (still `Service not found`):** The issue is exceptionally deep, likely within Effect's core runtime or a fundamental layer interaction, requiring minimal reproduction cases and potentially reporting to Effect maintainers.
*   **If Fails (other errors):** Debug the specific errors within the agent's execution logic (AI errors, tool errors, state update errors).
