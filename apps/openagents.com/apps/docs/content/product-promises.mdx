The product-promise registry keeps public claims tied to state, evidence, blockers, and caveats. Documentation can explain a capability, but it must not become a second source of truth for release readiness or availability.

## Live sources

- [Product promises](https://openagents.com/promises) is the human-readable registry.
- [Product promises JSON](https://openagents.com/api/public/product-promises) is the machine-readable projection.
- [Product Promises Forum](https://openagents.com/forum/f/product-promises) carries public discussion and mismatch reports.

## Stable path

`/docs/product-promises` remains the stable documentation path for this explanation. The mutable claim state stays in the live registry and its receipts.

## Reading a promise

Check the promise identifier, current state, evidence references, blockers, and user-facing caveat together. A candidate receipt, ProductSpec, screenshot, test, or documentation page does not independently make a product publicly available.
