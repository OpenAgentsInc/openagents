# Cloud Pylon Deployment Path

Status: M4 product runbook for Pylon v0.3 source RC cloud nodes.

This path runs a Pylon on a Linux VM the owner controls. The VM uses the
owner's OpenAgents identity and BYOK provider credentials, registers and
heartbeats like a local Pylon, and can pick up the owner's no-spend Autopilot
work while the laptop is closed.

It does not make the VM an OpenAgents-hosted runner, does not install platform
provider credentials, and does not grant settlement, payout, deploy, spend, or
Forum publication authority.

## Prerequisites

- Ubuntu 24.04 LTS or another systemd-based Linux host.
- Bun, Git, and a working outbound HTTPS path.
- An owner-granted OpenAgents agent token with assignment/work authority.
- BYOK Claude Agent readiness, either:
  - `ANTHROPIC_API_KEY`, or
  - a provider credential mode supported by the Claude Agent SDK.
- A VM user/operator who can run `sudo`.

The VM is the user's machine. Do not place OpenAgents platform credentials on
it. Store owner secrets only in the root-owned service env file generated by
the installer.

## Install

From the VM:

```sh
git clone https://github.com/OpenAgentsInc/openagents
cd openagents
sudo -E apps/pylon/scripts/install-cloud-node.sh
```

The installer accepts these environment variables:

```sh
export PYLON_REF=pylon.cloud.<owner>.<host>
export PYLON_DISPLAY_NAME="Owner Cloud Pylon"
export PYLON_RESOURCE_MODE=background_20
export OPENAGENTS_AGENT_TOKEN=<owner-granted-agent-token>
export ANTHROPIC_API_KEY=<owner-byok-key>
sudo -E apps/pylon/scripts/install-cloud-node.sh
```

Defaults:

- install dir: `/opt/openagents-pylon`
- Pylon home: `/var/lib/openagents-pylon`
- service user: `pylon`
- service name: `openagents-pylon`
- base URL: `https://openagents.com`
- control port: `4716`
- assignment worker loop: enabled with `PYLON_ASSIGNMENT_WORKER=1`

The service runs:

```sh
bun /opt/openagents-pylon/apps/pylon/src/index.ts node
```

with env from:

```text
/etc/openagents-pylon.env
```

That file is installed `0600` and may contain owner credentials. Never commit,
paste, or issue-comment its contents.

## Register And Verify

Check service health:

```sh
systemctl status openagents-pylon --no-pager
journalctl -u openagents-pylon -f
```

Verify local status:

```sh
sudo -u pylon env \
  PYLON_HOME=/var/lib/openagents-pylon \
  bun /opt/openagents-pylon/apps/pylon/src/index.ts status --json
```

Send a live heartbeat:

```sh
sudo -u pylon env \
  PYLON_HOME=/var/lib/openagents-pylon \
  PYLON_OPENAGENTS_BASE_URL=https://openagents.com \
  bun /opt/openagents-pylon/apps/pylon/src/index.ts presence heartbeat \
    --base-url https://openagents.com
```

Verify Claude Agent capability before leaving the node unattended:

```sh
sudo -u pylon env \
  PYLON_HOME=/var/lib/openagents-pylon \
  ANTHROPIC_API_KEY="$ANTHROPIC_API_KEY" \
  bun /opt/openagents-pylon/apps/pylon/src/index.ts provider go-online
```

The Pylon may declare `capability.pylon.local_claude_agent` only when the local
readiness probe reports ready. The probe checks credential presence and does not
print or persist credential values.

## Own-Job Worker Loop

The installer enables:

```text
PYLON_ASSIGNMENT_WORKER=1
PYLON_ASSIGNMENT_WORKER_INTERVAL_SECONDS=30
```

When enabled, `pylon node` continuously runs the no-spend assignment worker
loop against `PYLON_OPENAGENTS_BASE_URL`. It accepts only eligible no-spend
leases admitted by local gates. Paid leases remain blocked unless the wallet
send-readiness path explicitly proves readiness.

This is the cloud form of the M3 free own-Pylon lane:

- owner work placed on this Pylon is free and writes no buyer debit rows;
- SHC remains the metered fallback when this Pylon is offline, stale, or
  ineligible;
- browser `/autopilot/work` and `pylon work status` read the same work-order
  projection.

## Attach

From the VM:

```sh
sudo cat /var/lib/openagents-pylon/control-token
pylon attach http://127.0.0.1:4716 --token <control-token>
```

Do not expose the control port publicly. Use SSH port forwarding if attaching
from another machine:

```sh
ssh -L 4716:127.0.0.1:4716 <vm>
pylon attach http://127.0.0.1:4716 --token <control-token>
```

## Restart And Upgrade

Restart:

```sh
sudo systemctl restart openagents-pylon
```

Upgrade from `main`:

```sh
cd /opt/openagents-pylon
sudo git fetch --depth 1 origin main
sudo git reset --hard origin/main
sudo -u pylon env HOME=/var/lib/openagents-pylon PYLON_HOME=/var/lib/openagents-pylon bun install
sudo systemctl restart openagents-pylon
```

## Security Posture

- The VM belongs to the owner. OpenAgents does not control it.
- Use owner BYOK credentials only; no platform provider credentials.
- Keep `/etc/openagents-pylon.env` root-owned and `0600`.
- Keep the Pylon control API on loopback.
- Do not run a production treasury or payout mnemonic on this VM.
- Redaction law is unchanged: raw prompts, provider payloads, local paths,
  wallet material, and private repo content must not leave the VM.

On compromise:

1. Stop the service: `sudo systemctl stop openagents-pylon`.
2. Remove or rotate `/etc/openagents-pylon.env` secrets.
3. Revoke the owner-granted OpenAgents agent token.
4. Rotate the BYOK provider key or invalidate the provider session.
5. Deregister or quarantine the Pylon identity from OpenAgents operator tools.
6. Rebuild the VM from a clean image before re-registering.

## Acceptance Evidence

A complete M4 evidence bundle should include public-safe refs for:

- the cloud Pylon registration and fresh heartbeat;
- `capability.pylon.local_claude_agent` readiness;
- at least one owner work order selected for `requester_pylon`;
- an assignment lease accepted and closed by the cloud Pylon;
- the same work order visible through browser `/autopilot/work` and
  `pylon work status`;
- a no-debit funding projection for the own-Pylon lane.

The 24h unattended run is operational evidence, not a new authority surface.
Keep it as a receipt bundle or issue comment with refs only.
