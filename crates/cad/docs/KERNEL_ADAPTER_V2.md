# Kernel Adapter V2

Issue coverage: `VCAD-PARITY-011`

## Purpose

Finalize the `cad::kernel` v2 adapter contract for engine pluggability while preserving
v1 compatibility for existing primitive evaluation call sites.

## Contract Summary

`crates/cad/src/kernel.rs` now defines:

- `CadKernelAdapter` (v1, unchanged)
- `CadKernelAdapterV2` (descriptor + operation receipts)
- `KernelAdapterV2Descriptor` (engine metadata + capabilities)
- `KernelAdapterV2Bridge<K: CadKernelAdapter>` (v1 -> v2 bridge)
- `KernelAdapterRegistry` (engine registration + active-engine switching)

The default OpenAgents descriptor is provided by:

- `openagents_kernel_adapter_v2_descriptor()`

## Parity Artifact

- `crates/cad/parity/kernel_adapter_v2_manifest.json`

Generated by:

cargo run -p openagents-cad --bin parity-kernel-adapter-v2
```

Fixture check gate:

scripts/cad/parity-kernel-adapter-v2-ci.sh
```

## Determinism Contract

- Descriptor capabilities are sorted/deduplicated via `normalized()`.
- v2 operations emit deterministic receipts (`KernelOperationReceipt`) including tolerance and capability-count diagnostics.
- `KernelAdapterRegistry` enforces stable engine IDs and explicit active-engine switching.
- `crates/cad/tests/parity_kernel_adapter_v2.rs` verifies fixture equivalence.
