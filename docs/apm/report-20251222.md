# APM Historical Report - December 22, 2025

> **STATUS: COMPLETE** - Data collected from both macOS and Linux machines.

## Methodology

APM (Actions Per Minute) measures agent velocity using the formula:

```
APM = (messages + tool_calls) / duration_minutes
```

**APM Tiers:**
| Range | Tier | Description |
|-------|------|-------------|
| 0-5 | Baseline | Human-in-the-loop, lots of thinking time |
| 5-15 | Active | Steady work pace |
| 15-30 | Productive | Efficient autonomous execution |
| 30-50 | High Performance | Fast parallel tool usage |
| 50+ | Elite | Maximum efficiency |

## Observations

### Key Findings

1. **~237K interactive actions tracked** - Combined across both machines over ~25 days, representing ~583 hours of Claude Code activity.

2. **Linux shows 2x higher APM than macOS** (11.2 vs 5.6):
   - Linux uses actual timestamp-based duration calculation
   - macOS uses estimated 15 min/session (likely underestimating duration)
   - The difference may be methodological rather than real

3. **Autopilot metrics are unreliable**:
   - rlog files only capture tool calls, missing messages
   - Subagent JSONL captures short-lived exploration tasks separately
   - No unified view of full autopilot activity exists

4. **Data collection needs improvement**:
   - Need consistent logging across all session types
   - Need actual timestamps instead of duration estimates
   - Need to capture full message + tool activity for all modes

### Methodology Notes & Limitations

**Data Source Inconsistency (CRITICAL):**

The different data sources capture fundamentally different things and **cannot be meaningfully compared**:

| Source | What it captures | Avg Duration | Tool Logging |
|--------|------------------|--------------|--------------|
| UUID JSONL | Full interactive sessions | ~54 min | Complete (msgs + tools) |
| Agent JSONL | Short subagent tasks | ~23 sec | Complete (msgs + tools) |
| rlog | Main autopilot loop | ~47 min | Incomplete (tools only, no msgs) |

**Why APM comparisons are problematic:**
1. **rlog files don't log messages** - only tool calls, undercounting actions
2. **Agent JSONL files are subagent tasks** - spawned by autopilot for search/exploration, very short-lived
3. **No single source captures full autopilot activity** - main loop in rlog, subagents in JSONL

**Duration Calculation:**
- Interactive (UUID): Actual timestamps from first to last entry in file
- Autopilot: File modification time minus @start timestamp (rlog) or first/last timestamp (JSONL)

**Recommendation:**
Until we have unified logging that captures all autopilot activity (main loop + subagents) in a consistent format, APM comparisons between interactive and autopilot modes are unreliable.
