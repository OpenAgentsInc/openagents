## 1. What the episode says

Episode 243 opens with the thesis — "we are now in the inference business" — and
sets one goal for the session: get Khala into a real coding tool (OpenCode,
chosen as "probably the best open source harness... Extensible, and they use
Effect") and watch the public tokens-served counter climb on real work. Over the
~two-hour session the counter moves from ~1M to ~13M served, ~12M in-video,
ending with ten concurrent OpenCode sessions all routing through Khala.

The integration surfaces two production blockers, fixed live. First, Khala's
OpenAI-compatible route rejected what OpenCode actually sends: message content
arrays plus streamed tool-call deltas ("finish_reason: tool_calls with no
visible tool calls array"). The fix lands across both the Hydralisk and
Fireworks adapters — request mapping, non-streaming responses, and SSE deltas —
with the stated contract that "every adapter that speaks OpenAI-ish chat should
preserve that field, instead of each lane having a slightly different failure
mode." Second, the Autopilot desktop's "Talk to Khala" returned
`model_unavailable` from a stale `khala-mini` slug, re-pointed to the single
`openagents/khala`.

Two engineering threads run alongside. The public counter is converted to
WebSocket push, which exposes a backward-jumping double-count: the display
oscillates between the true ledger total (~8M) and an overcount (~15–21M). Root
cause is layered — a seed-plus-replay double count, then a "fix" that silently
never deployed (a SIGKILL-137, then a `wrangler`-not-found 127 hidden behind an
exit-zero wrapper) — resolved by an authoritative-total monotonic reducer: the
server carries the running total on every event, the client takes
`max(displayed, total)`, never sums deltas. Separately, the free tier (200
requests / 200k tokens — "rookie numbers") is raised on a real cost model with
provider analytics, the provider mix is named (Fireworks DeepSeek V4 Flash,
Gemini Flash, OpenRouter free, own-infra GPT-OSS 20B/120B, and the GLM-5.2 REAP
lane from Z.ai), and the GLM lane gets a speed win via MTP-2 speculative
decoding with `min_p` dropped (~35 → ~48 tok/s).

The strategy is read verbatim into the record from the "Khala Inference GTM
Push" doc — deliberately, "because agents are going to read the transcripts of
my talks." Three pillars: dogfood (route everything internal through Khala, QA
first, internal tokens real but distinguishable from external demand); ecosystem
tools (be a one-config-line drop-in; OpenCode first, then Aider/Cline/Vercel AI
SDK); benchmarking (build "the gym" — not "a gem," a running transcription joke
— benching Khala against BigPickle, OpenCode's default free model, then open,
then paid models). North Star: "tokens served per day, bent exponential —
honestly."

The concurrency demo itself is instructive in what broke: the first five-session
fanout had four of five sessions fail with invalid-request errors under
multi-step planning; a fixed gateway was live on `origin/main` but production
was still serving old behavior until redeploy; one OpenCode session was killed
with code 143 by the surrounding tool-execution layer when it got compacted; a
402 appeared mid-run from unfunded credits; and the operator repeatedly could
not see or steer what a harness was doing ("I'm so excited to get away from
these busted-ass harnesses onto our own things, so that we can control them").
The wrap: ten instances running, 13M crossed, "probably we'll build the gym
next."

## 2. Where it stands against Revision 31

The single biggest change: OpenCode's role inverted. Episode 243 treats OpenCode
as the integration target and first ecosystem landing; under Revision 31 it is
the parity benchmark and teardown subject. Three OpenCode teardowns (desktop
app, V2 architecture, Effect architecture) now inform the greenfield OpenAgents
Desktop (`apps/openagents-desktop`, Electron + Effect Native, tokenless renderer
behind a host-owned Runtime Gateway), and R5 is defined as the "practical
OpenCode-parity Desktop workbench," with a 20-area parity audit standing at 1
landed / 6 partial / 3 scaffold / 10 absent. The episode's own closing
frustration — wanting off "busted-ass harnesses onto our own things" — is the
seed of that inversion; the transcript predicts its own supersession. Notably,
OpenCode moved 109 commits in the five days after the audit pin, confirming why
building on it as a shell was untenable.

The gateway work aged well. The tool-call/content-array fixes and the "one
contract, every adapter preserves it" rule are now embedded in the closed
FC-BRAIN #8600 Khala inference gateway (exact receipts, session/turn caps, typed
fallback). The `khala-mini` fix became settled vocabulary: single model surface
`openagents/khala`, variant names deprecated. The counter discipline the
double-count saga forced is now doctrine: the `khala-tokens-served` endpoint is
a projection, "counter movement is never proof," and every public number
reconciles to exact `token_usage_events` rows.

Concurrency moved from shell-spawning to substrate. Ten OpenCode processes
sharing one coordinator is superseded by the fleet substrate: FC-1 #8637 durable
FleetRun authority, FC-2 #8633 standing mixed Codex/Claude/Grok Pylon supervisor
with one claim registry, FC-3 #8639 supervision/steer/approval/receipts, and
#8640 (open) the live simultaneous Codex+Claude proof. Typed Khala coding
delegation through Pylons (`--workflow codex_agent_task`, isolated accounts,
typed `account_exhausted`/`account_rate_limited` failure classes) is the current
form of "send coding work through Khala." The gym remains direction: the GTM doc
itself labeled gym-to-Khala "not a live claim," and no shipped recurring
head-to-head exists; the benchmarking intent lives on in Eval Suite vocabulary
and the fixture/owner-armed lane split the episode read aloud.

## 3. What to extract and carry forward

- **The adapter contract is the one-engine argument in miniature.** "Carry tool
  calls once; every OpenAI-ish adapter preserves the field" generalizes into the
  canonical architecture: one engine behind a versioned typed protocol
  (Thread→Turn→Item + Work Unit/Receipt), so lane-specific failure modes cannot
  exist by construction.
- **The counter saga is the three-read-surfaces lesson.** The double count came
  from treating a volatile event stream as additive authority. The durable
  pattern — server-side authoritative total, monotonic max reducer,
  snapshot-plus-cursor, dedupe by ref — is exactly the split now canonized as
  bounded projection / durable replay log with sync marker / volatile stream.
- **"It never actually deployed" justifies the proof rungs.** The exit-zero
  wrapper hiding a dead `wrangler` deploy, and prod serving stale behavior while
  `origin/main` was fixed, are the concrete cases behind separating code-landed
  from deployed from live-proven. A green local check is rung one, not rung
  four.
- **Concurrency failures name the substrate requirements.** Four-of-five
  sessions dying under multi-step planning, the code-143 kill from the
  surrounding tool layer, the mid-run 402, and "don't just stop stuff — tell me
  what happened" translate directly into durable admission before execution,
  typed failure classes instead of generic session death, and "a timeout is not
  an outcome."
- **Steering pain specifies the workbench.** Not being able to message a running
  harness, escape keys doing the wrong thing, output invisible during long
  pushes — these are requirements for steer-vs-queue delivery and the
  supervision surfaces of FC-3, and part of why R5 is a parity workbench rather
  than an integration.
- **The honesty bar from the GTM doc is keepable verbatim.** Internal dogfood
  tokens real but distinguished from external demand; fixture numbers labeled
  illustrative, never published as measurements; direction labeled direction
  until it has a green promise.

## 4. How to make it come true

The episode's real promise — "many concurrent coding sessions doing meaningful
work through our own surface, measured honestly" — is carried by the current P0:
reliable Desktop/mobile coding and fleet software, with remote-first portable
sessions. Concretely: land #8640 (live simultaneous Codex+Claude proof) to make
mixed-provider concurrency live-proven under the FC-2 claim registry rather than
demo-proven under one terminal; run sustained multi-worker burns through
FleetRun authority (FC-1) with per-turn exact token rows and closeout receipts
as the durable replacement for watching the counter climb; and advance the
20-area OpenCode parity audit toward R5 so the "ten sessions, supervised"
experience exists inside OpenAgents Desktop with the Runtime Gateway holding
tokens.

The gym is the one unbuilt commitment worth a bounded leaf: a recurring
Khala-vs-baseline coding evaluation (BigPickle-class free model first, then
open, then paid) built on the existing fixture-driven harness — deterministic
fixture lane labeled illustrative, owner-armed real lane producing the only
publishable numbers, cost per accepted outcome and P50/P90/P99 latency as axes.
Described without inventing issue numbers; it should enter the roadmap as an
Eval Suite, not a standalone product.

Acceptance in six-rung vocabulary: mixed-provider concurrency reaches
live-proven at #8640 and owner-accepted only after a sustained supervised run
with reconciled receipts; parity areas move absent → scaffold → landed against
the audit; any published benchmark claim requires the owner-armed real lane
(deployed + live-proven) before it appears outside internal docs; the
tokens-served curve is never itself acceptance evidence — reconciliation to
`token_usage_events` is.

## 5. Risks and tensions

- **Do not build on an unfrozen third-party client.** OpenCode's
  109-commits-in-5-days pace after the audit pin is the standing argument: it is
  a benchmark and teardown subject, never the shell. Any renewed "just integrate
  with X" impulse must clear that bar.
- **Scale demos are not proof.** Ten sessions and a climbing counter felt like
  "mission accomplished," but the session itself showed the counter overcounting
  2x while the demo ran. The invariant — counter movement is never proof —
  exists because of this episode; never regress to odometer-as-evidence.
- **The subscription-resale line must stay bright.** The episode's opening muses
  about connecting Claude/Codex tokens into Khala and converting "subsidized
  usage" to Bitcoin. The transcript itself draws the line ("you can't resell
  inference for some things with subs, but you can absolutely have your agent do
  work for money"), and current policy hardens it: owner-local subscription
  capacity is never resold; own-capacity agent work with exact receipts is the
  only compliant form.
- **Deadline-driven gate skipping.** "Skip the full check... generally I
  shouldn't say that, but I'm going to do it right now" is exactly the behavior
  the six rungs and behavior-contract oracles exist to prevent; a skipped gate
  during a demo is how "never actually deployed" happens.
- **Deprecated vocabulary.** `khala-mini` (and any variant naming) is dead;
  "gem" for gym is a transcription artifact already corrected in-corpus;
  BigPickle references should stay scoped to OpenCode's free-model baseline, not
  treated as a maintained comparison target without re-verification.
- **Dogfood inflation.** The North Star is "bent exponential — honestly."
  Internal fanout can move the counter arbitrarily; the internal/external demand
  split in analytics is a permanent obligation, not a launch-week nicety.
