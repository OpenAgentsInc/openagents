# docs/fable/episodes

Per-episode Fable analyses of the OpenAgents video corpus, working backward
from the most recent episode. Each file reads one transcript in
`docs/transcripts/`, then answers the same five questions against the current
roadmap: what the episode says, where it stands against
`docs/sol/MASTER_ROADMAP.md` (Revision 31 at time of writing), what to
extract and carry forward even where specific directions were superseded, how
to make it come true under the current gate structure, and what risks or
superseded framings must not be carried forward.

These are analyses, not roadmap authority. Sequencing lives in the Sol master
roadmap. Factual status lives in code, live issues, and receipts. Transcripts
are machine-generated — verify wording against the video before quote-grade
use.

## Conventions

Each analysis pins its date, author (Fable), and source transcript, and uses
the shared five-section template. Follow-on episodes should extend this
directory one file per episode with the same structure, and update this
index. When an analysis identifies work worth dispatching, it names existing
issues/gates or describes a bounded new leaf — it never invents issue
numbers.
