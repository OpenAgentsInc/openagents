## One arc users climb

```text
Tool (dispatch work)
  → Employees (standing, named, budgeted)
    → Brain (governed knowledge you own)
      → Placement (hosted → BYO model → private → Reactor)
        → Network (receipts → reputation → routed work → settlement)
```

## Horizons (content) vs phases (execution)

Horizons (H0–H6) describe product maturity. Phases (P0–P7) order
*shipping*. MASTER_ROADMAP owns phase order.

| Phase | Focus | Horizon weight |
| --- | --- | --- |
| P0 | Mobile MVP tested + store artifacts | H0 |
| P1 | Sales agent + landing + outbound | distribution |
| P2 | BYO Codex daily-driver on agent computers | H1 |
| P3 | Standing employees | H2 |
| P4 | Employees + company brain | H3 |
| P5 | Templates + integrations | H4 |
| P6 | Trust layer (skills, canaries, ceilings) | H5 |
| P7 | Scale / suite GTM / network density | H6 |

Testing is not a phase that ends: each lane exits with Eval Suite green
at target ladder rungs.

## Non-negotiable spine rules

1. One account across surfaces.
2. One credits ledger; many fill rails (IAP, Stripe, crypto/Lightning,
   sales grants).
3. One Khala Sync data plane — no screen-scraping between apps.
4. One Agent Computers execution substrate for cloud work.
5. One promise registry for public claims.
6. Programmatic access: anything on the dashboard is a typed API/tool
   target so the suite manages the suite.

## Desktop vector (remember)

- **Do:** agent console + fleet + inbox + verification instruments.
- **Don't:** VS Code feature war as product identity.
- Editor is the manager's magnifying glass.

## Mobile vector (remember)

- Productive without desktop forever.
- Pairing is upgrade, never requirement.
- Front-door metric: discovery → first useful agent outcome in one sitting.

## Services and product

Same fleet fulfills:

- Self-serve credits usage
- Fixed-scope accepted outcomes
- Standing retainers / "on autopilot" style packages

Verticals = connectors + corpus + verification rubric. Never forks.
