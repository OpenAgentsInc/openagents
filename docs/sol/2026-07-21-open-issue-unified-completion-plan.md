### Managed sandbox Phase 2 admission

This plan activates #9032 because SBX-09 is complete.
The Phase 2 design must keep these rules:

- A snapshot is a verified content checkpoint. It is not a VM memory image.
- A checkpoint binds the source sandbox, generation, image, toolchain,
  repository post-image, digest, size, retention, and omission facts.
- A checkpoint contains no credentials, provider state, process memory, PTY,
  socket, port, or network identity.
- A fork creates a new sandbox identity, generation, grants, and cleanup duty.
- Private ingress uses a short owner and audience scoped capability.
  It has a short expiry, explicit revoke, an audit record, and cleanup proof.
- Public VNC, an ungated preview URL, and a permanent ingress route are not
  allowed.
- Google Cloud remains the only managed runtime and storage authority.

### Managed RLM admission

This section is the separate managed boundary required by #9144.
It admits implementation after a bounded claim.

- The engine remains `@openagentsinc/ai/rlm` and `@openagentsinc/rlm`.
  OpenAgents does not add a second engine or arbitrary code executor.
- A managed corpus source can read only current owner-scoped workroom and
  evidence data in the admitted Google Cloud custody domain.
- Owner-local corpus data stays owner-local. The adapter cannot copy it to a
  managed corpus.
- The source adapter resolves owner, corpus, policy, and source generation
  before it reads data. A model cannot widen this scope.
- The adapter does not create a second raw corpus copy. It uses authorized
  bounded reads from the source of record.
- An admitted model receives only the bounded source parts that the current
  request needs. Current model and account policy controls this transfer.
- Semantic mode is off by default. Each run needs an explicit typed budget.
  One run can use at most USD 5. The program can use at most USD 25 each day.
  The lower limit always applies when another authority limit is smaller.
- `RlmArtifactSink` is off by default. An explicit run can enable it for an
  owner-private evidence pack.
- The service gives each artifact a content address and encrypts it under the
  current owner-scoped Google Cloud storage policy. It records lineage, media
  type, exact bytes, and a deletion state.
- Default artifact retention is 24 hours. A typed owner choice can select 7
  or 30 days. Expiry and manual delete remove the object and its access grant.
- Source revocation prevents new reads. Artifact deletion prevents new reads
  and produces a deletion receipt.
- Logs and public receipts contain refs, counts, digests, and bounded failure
  facts. They contain no raw corpus, prompts, model output, or private paths.
- Recall and evidence-pack output stays an untrusted cited candidate.
  It cannot verify work or grant release, acceptance, payment, or a promise.
- Production availability and public quality claims remain outside #9144.
