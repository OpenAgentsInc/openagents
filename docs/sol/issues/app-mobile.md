# APP-MOBILE: OpenAgents mobile — Sarah home and fleet supervision on Effect Native

## Outcome

The current Khala Code mobile app becomes **OpenAgents**, one iOS/Android app
with Sarah as its home relationship and coding-fleet supervision as the first
deep capability.

## Scope

1. Preserve the shipping sign-in→repo→work→writeback→credits path during
   migration.
2. Reframe navigation and product identity as OpenAgents; Sarah conversation,
   voice, Blueprint, active runs, approvals, and receipts are the home flow.
3. Keep direct account connection/settings for expert recovery while Sarah can
   guide the same typed actions.
4. Reauthor every retained screen on the Effect Native mobile renderer; use
   typed navigation, gestures, virtualization, and foreign hosts for native
   STT/Apple FM modules.
5. Use Khala Sync for conversation and fleet continuity across web, mobile,
   and desktop.
6. Delete converted React/khala-primitive screens and retire the Khala Code
   product name.
7. Preserve owned OTA, local build/submission, QAM, Maestro, and behavior-
   contract gates.

## Exit

A Sarah conversation and active multi-stream run started on web appear live in
OpenAgents mobile; a mobile steer/approval changes the same run and appears in
OpenAgents Desktop. Both platforms render through Effect Native, launch gates
remain green, and the app ships under the OpenAgents product identity.
