# OpenAgents Systems Index

This directory is the source-of-truth master list for OpenAgents system
documentation. Each row below identifies a real system, its purpose, the owning
code paths, current doc status, and any related architecture record. This pass
is scaffold-only: rows start as `stub` until a dedicated per-system document
lands.

## Process

1. Claim a system by adding a doc at `docs/systems/<system-slug>.md` using
   [`_template.md`](./_template.md).
2. Keep the row in this index. Update its doc status from `stub` to
   `documented` only when the per-system doc covers responsibilities,
   boundaries, code paths, data model, interfaces, dependencies, invariants,
   verification, and open questions.
3. Link applicable records under `docs/adr/` if an ADR directory or system ADR
   exists. If there is no ADR, keep `None yet`.
4. Before changing authority, routing, payment, projection, or public-claim
   surfaces, read [`/INVARIANTS.md`](../../INVARIANTS.md), the relevant
   app-local invariant ledger, and the system doc.
