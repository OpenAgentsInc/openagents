## Learning by Construction

A circular diagram titled "Learning by Construction" shows the following flow:
- "verified program" → "module" → "library grows" → "compose modules" (back to "verified program").
- To the right, an upward arrow is labeled "more capability over time."
- Below, the text says "... like an agentic npm."

"So ChatGPT wants me to tell you that this is called 'learning by construction.' We have a verified program — a module, a thing that can go into a transformer but in a way that's modular, so it can be combined with other modules. It's like the idea of npm. There's no npm for agents, right? Everyone's building their own thing. But with npm, you have a registry of a bunch of different pieces of software, packages, that compose other packages. That same idea for computation — building programs out of modules — that's how conventional software works. That library of modules is going to grow into a pretty powerful registry: more powerfully composed modules that go back into programs, and then we get more capability over time. So it's like an agentic npm — but now imagine we have the opportunity to do things beyond what npm did, like preemptively solve for some of the security and trust constraints by attaching better cryptographic verification or payments. More on that in an upcoming video as we reboot our plugin marketplace from a while back."
