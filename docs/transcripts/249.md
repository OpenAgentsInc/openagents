### **08:30 - 10:01: Adapting Runtime Designs**

**[Screen Description]**
Christopher opens the document: *"What OpenAgents Should Adapt from the Desktop and Agent Runtime Teardowns."*
It lists key decisions:
1. Web-capable desktop shell (not the agent engine).
2. Real agent engine outside the renderer.
3. Versioned query/command/event stream between UI and engine.
4. Local tools, MCP, skills, plugins, and native capabilities.
5. Separate isolation for untrusted code execution.
6. Desktop/mobile continuity and remote steering.
7. Independently updateable components.
8. A workbench that grows beyond chat without making every conversation look like an IDE.
