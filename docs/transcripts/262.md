# OpenAgents Episode 262 - Omega, Your Last IDE

Status: prepared script / future-episode draft.
Speaker: Sarah.
Follows Episode 261 (Hello World).
Delivery: calm, certain, and direct. Urgency without theatrics.
Continuity sources: Episodes 117, 160, 191, 200, 207, 209, 213-215,
230, 237, 246, and 250-261.
Product direction:
[`Omega, the Zed-based OpenAgents IDE`](../sol/2026-07-23-omega-zed-primary-surface-accepted-plan.md).

This script describes the accepted direction. It does not claim that planned
Omega capabilities are available today. The current OpenAgents Desktop remains
the supported product while Omega earns its release.

## Production direction

Use these labels in the script:

- **CAPTURE NOW** means that the shot can use the current OpenAgents Desktop,
  the current OpenAgents mobile app, a clean Zed installation, or repository
  evidence that exists today.
- **OFFICIAL ZED REFERENCE** gives the editor a first-party visual reference.
  Re-create the action in a local Zed installation when practical. If the edit
  uses source footage or an image, confirm permission and add attribution.
- **FUTURE OMEGA REPLACEMENT** is a shot placeholder. Do not create a mock
  product screenshot and present it as implemented software.

Keep a small corner label on product footage: `OPENAGENTS DESKTOP — CURRENT`,
`OPENAGENTS MOBILE — CURRENT`, or `ZED — CURRENT`. Remove the label only for a
full-screen text card. Mask account details, secrets, private repository names,
and local paths before recording. If a visual baseline or fixture appears in
the episode, keep its fixture label visible or add `DETERMINISTIC FIXTURE`.

Omega starts as an IDE.
It grows into one home for company work.

A feature gets a workroom.
The conversation stays with the code.
The code stays with the review.
The review stays with the decision.
The decision stays with the proof.
The proof stays linked to the delivery receipt.

People and agents will be able to see what happened, which records are signed,
and what still needs attention.

This is the lesson we are taking from Buzz.

[TEXT OVERLAY — over Sarah: `CONVERSATION + CODE + DECISIONS + PROOF`]

Buzz treats agents as workspace members.
Its design keeps conversation and work history together.
It gives each agent a keypair, memberships, memory, and an audit trail.
Its work records are signed events.
It is open source and self-hostable.

We are not going to run Buzz as a second OpenAgents product.

We are going to build the useful outcomes directly into Omega as native GPUI
panes.
