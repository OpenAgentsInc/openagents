## Editorial source notes

The archive shows real products and real releases.
Do not frame the history as a record of vaporware.
The honest weakness is the lack of one durable shared product foundation.

Episode 183 already introduced another zero-base reset.
Episode 196 already argued for one consolidated agent product.
Episode 251 deliberately narrowed the product to a useful Desktop base hit.
Episodes 256-259 then showed why the durable surface must be a real IDE.

Omega differs from earlier resets.
It does not start from an empty folder.
It starts from a mature open-source editor.

Cursor first used CodeMirror.
Its version 0.2.0 moved onto a VSCodium fork.
Cursor later summarized its strategy as forking VS Code instead of making an
extension.
Use Cursor's
[`0.2.0 changelog`](https://cursor.com/changelog/0-2-0) and
[`Meet the new Cursor`](https://cursor.com/blog/cursor-3) as first-party
sources.

Use the local Zed application for product footage.
Keep `ZED - CURRENT` visible.
The public Omega repository proves that the fork exists.
It does not prove that a branded Omega application exists.
